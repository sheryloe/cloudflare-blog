## 왜 이 구조로 가나

기존의 공개 웹, 관리자, API 중심 구조는 빠르게 시작하기에는 좋았지만 글 작성과 배포 경계가 자주 섞였습니다.

새 구조에서는 작성은 로컬에서 끝내고, 배포는 `apps/blog-web` 하나만 담당합니다.

## 지금 확인할 수 있는 것

- `/ko/`, `/en/` 언어 홈
- 태그와 카테고리 정적 페이지
- 언어 전환 링크
- frontmatter 기반 정적 빌드

## 다음 단계

로컬 작성기에서 번역과 점수를 확인한 뒤 MDX를 복사해 넣는 흐름을 완성하면, 실제 게시 작업도 훨씬 단순해집니다.
