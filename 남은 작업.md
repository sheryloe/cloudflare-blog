# 남은 작업

## 현재 기준

현재 저장소는 아래 범위까지는 구현되어 있습니다.

- `apps/web`, `apps/admin`, `apps/api`, `packages/shared` 분리 완료
- `pnpm` 워크스페이스 전환 완료
- 공개 웹 기본 페이지와 관리자 앱 기본 화면 구현 완료
- 관리자 로그인, 글 CRUD, 카테고리/태그 관리, R2 업로드 구현 완료
- 공개 홈, 글 상세, 카테고리/태그 아카이브 구현 완료
- 공개 웹/관리자 앱 UI 1차 Tailwind 정리 완료
- 배포 경계 하드닝과 CORS/auth 보강 완료

아래 항목들은 아직 남아 있는 작업입니다.

## 1. 운영 시작 전 필수 작업

### 1-1. Cloudflare 실제 배포 설정

- `apps/web`, `apps/admin`, `apps/api`를 각각 Cloudflare에 실제로 연결
- Pages 환경변수 설정
  - `apps/web`: `VITE_API_BASE_URL`, 필요 시 `VITE_ADMIN_APP_URL`
  - `apps/admin`: `VITE_API_BASE_URL`, 필요 시 `VITE_PUBLIC_APP_URL`
- Worker 환경변수/secret 설정
  - `PUBLIC_APP_ORIGIN`
  - `ADMIN_APP_ORIGIN`
  - `ADMIN_EMAIL`
  - `ADMIN_PASSWORD_HASH`
  - `JWT_SECRET`
  - `R2_PUBLIC_BASE_URL`
- 실제 도메인을 같은 eTLD+1 아래로 배치
  - 예: `blog.example.com`, `admin.example.com`, `api.example.com`

### 1-2. 관리자 인증 실사용 검증

- 실제 `ADMIN_PASSWORD_HASH` 생성 후 로그인 확인
- 로그인, 세션 유지, 로그아웃 흐름 실배포 환경에서 확인
- 허용되지 않은 origin에서 관리자 API 차단되는지 확인

### 1-3. 기본 콘텐츠/데이터 준비

- 최초 카테고리/태그/샘플 포스트 입력
- R2 업로드 후 공개 글의 커버 이미지 연결 확인
- 관리자에서 발행한 글이 공개 홈과 상세 페이지에 정상 노출되는지 확인

## 2. MVP에서 아직 비어 있는 기능

### 2-1. 공개 페이지 placeholder 제거

- `/about` 실제 콘텐츠 페이지로 전환
- `/search` 실제 검색 기능으로 전환
- `/rss.xml` 실제 RSS 생성으로 전환
- `/sitemap.xml` 실제 sitemap 생성으로 전환

### 2-2. 검색 기능 구현

- Worker에 검색용 API 추가
- 제목/본문/태그 기준 검색 동작 정의
- 결과가 많은 경우 정렬/제한 규칙 정리
- 공개 웹 `/search` UI와 연결

### 2-3. RSS / sitemap 생성

- 발행된 글 기준 RSS XML 생성
- 공개 페이지 기준 sitemap 생성
- 배포 후 크롤러가 읽을 수 있도록 경로와 응답 헤더 확인

## 3. 관리자 앱 품질 보강

### 3-1. 글 편집 경험 개선

- Markdown 작성 보조 UI 추가
- 미리보기 또는 split view 추가
- slug 자동 생성/수정 UX 개선
- 저장 중/실패 상태 표시 강화

### 3-2. 미디어 관리 개선

- 업로드 후 자산 검색/필터링
- 자산 삭제 또는 비활성화 정책 추가
- 커버 이미지 선택 UX 개선
- alt text 편집/수정 기능 추가

### 3-3. 목록 관리 기능 보강

- 글 목록 정렬/필터링/검색
- 카테고리/태그 사용 수 표시
- 긴 목록 대비 pagination 또는 lazy loading

## 4. 문서 모델은 있지만 아직 안 붙은 기능

- 공개 웹 `/series/:slug`
- 관리자 `/series`
- Worker series 조회/관리 API

### 4-2. Related posts

- 글 상세 하단 관련 글 추천 규칙 정의
- 카테고리/태그/series 기반 추천 방식 결정
- 공개 글 상세 페이지에 연결

### 4-3. 예약 발행

- 관리자 실패 요청 로그 확인 방식 정리

## 메모

- 현재 구조를 다시 갈아엎을 필요는 없습니다.
- 다음 단계는 재스캐폴드가 아니라 기능 완성과 운영 준비입니다.
- 우선순위는 `실배포 가능 상태 만들기` -> `placeholder 제거` -> `CMS 고도화` 순서가 가장 안전합니다.
